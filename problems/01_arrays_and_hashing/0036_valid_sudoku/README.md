# 36. Valid Sudoku

**Difficulty**: Medium  **URL**: https://leetcode.com/problems/valid-sudoku/

## 題目描述（中文）

判斷一個 9 × 9 的數獨棋盤**目前的填寫狀態**是否合法。需符合：

1. 每**列**包含 1-9 各最多一次。
2. 每**行**包含 1-9 各最多一次。
3. 每個 3 × 3 的**子方格**包含 1-9 各最多一次。

注意：
- 棋盤可能含未填的格子，以 `'.'` 表示。
- 只需驗證**已填**的數字是否合法，不需判斷是否可解。

## 範例 / 限制

**範例 1**
```
Input:
[["5","3",".",".","7",".",".",".","."]
,["6",".",".","1","9","5",".",".","."]
,[".","9","8",".",".",".",".","6","."]
,["8",".",".",".","6",".",".",".","3"]
,["4",".",".","8",".","3",".",".","1"]
,["7",".",".",".","2",".",".",".","6"]
,[".","6",".",".",".",".","2","8","."]
,[".",".",".","4","1","9",".",".","5"]
,[".",".",".",".","8",".",".","7","9"]]
Output: true
```

**範例 2**：將上面 board[0][0] 改為 `"8"`（與 [3][0]="8" 同行衝突），輸出 `false`。

**限制**
- `board.length == 9`
- `board[i].length == 9`
- 每格為一位數字 `'1'-'9'` 或 `'.'`

## 解法

### 解法一：三趟分別檢查
分別走訪所有列、行、子方格，每趟用集合判重。
- **時間複雜度**：O(81) = O(1)（固定大小）
- **空間複雜度**：O(1)

### 最佳解：一遍走訪 + 三組集合

**核心思路**：
- 用三個 dict（或長度 9 的陣列）：`rows[i]`、`cols[j]`、`boxes[b]`，每個值是該列/行/子方格已見過的數字集合。
- 子方格索引 `b = (i // 3) * 3 + (j // 3)`。
- 走訪每格 `(i, j)`，若 `c == '.'` 跳過；否則檢查 `c` 是否已存在於 `rows[i] ∪ cols[j] ∪ boxes[b]`，是則回傳 `false`，否則加入三個集合。

- **時間複雜度**：O(1)（81 格固定）
- **空間複雜度**：O(1)

## 範例追蹤（最佳解）

```
走到 (0,0)='5'：rows[0]={5}, cols[0]={5}, boxes[0]={5}
走到 (0,1)='3'：rows[0]={5,3}, cols[1]={3}, boxes[0]={5,3}
... 持續直到結束未發現衝突 → true ✓
```

若改 (0,0) = '8'：
```
走到 (3,0)='8'：cols[0]={5,8} ... 後續到 (3,0) 時 cols[0] 已有 '8'? 反向：
走到 (0,0)='8' → cols[0]={8}
走到 (3,0)='8' → 8 in cols[0] → 衝突，回傳 false ✓
```

## 關鍵點 / 易錯陷阱

- 子方格索引：`(i // 3) * 3 + (j // 3)`，0~8。
- `'.'` 必須跳過，不可加入集合。
- 三個集合都要檢查，缺一不可。
- 一次走訪可同時驗證三類規則，比三趟更精簡。
