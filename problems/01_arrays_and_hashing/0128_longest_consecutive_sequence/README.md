# 128. Longest Consecutive Sequence

**Difficulty**: Medium  **URL**: https://leetcode.com/problems/longest-consecutive-sequence/

## 題目描述（中文）

給定一個未排序的整數陣列 `nums`，找出**最長連續整數序列**的長度。  
連續序列定義：例如 `[1, 2, 3, 4]` 長度為 4，元素值連續但**不要求在原陣列中相鄰**。

**進階**：演算法時間複雜度需為 O(n)。

## 範例 / 限制

**範例 1**
```
Input:  nums = [100,4,200,1,3,2]
Output: 4
解釋：最長連續序列是 [1,2,3,4]
```

**範例 2**
```
Input:  nums = [0,3,7,2,5,8,4,6,0,1]
Output: 9
解釋：[0,1,2,3,4,5,6,7,8]
```

**限制**
- `0 <= nums.length <= 10⁵`
- `-10⁹ <= nums[i] <= 10⁹`

## 解法

### 解法一：排序後線性掃描
排序後逐一比較，計算最長連續長度。
- **時間複雜度**：O(n log n)
- **空間複雜度**：O(1) 或 O(n)（取決於排序實作）

### 最佳解：HashSet + 從序列起點展開

**核心思路**：

1. 把所有數字放入 `set` 以便 O(1) 查詢。
2. 對每個 `x ∈ set`，**只在 `x` 是序列起點時才開始計算**：起點的條件是 `x - 1 ∉ set`。  
3. 從 `x` 開始，向上連續查找 `x+1, x+2, ...`，計算長度。
4. 維護全域最大長度。

關鍵：每個元素**只會被檢查兩次**（一次判斷起點，一次被某個起點往上展開），因此整體 O(n)。

- **時間複雜度**：O(n)
- **空間複雜度**：O(n)

## 範例追蹤（最佳解）

```
nums = [100, 4, 200, 1, 3, 2]
set  = {100, 4, 200, 1, 3, 2}

x=100: 99 ∉ set → 起點。展開：100 → 停。長度 = 1
x=4:   3 ∈ set → 非起點，跳過。
x=200: 199 ∉ set → 起點。展開：200 → 停。長度 = 1
x=1:   0 ∉ set → 起點。展開：1, 2, 3, 4 → 停。長度 = 4 ✓
x=3:   2 ∈ set → 跳過。
x=2:   1 ∈ set → 跳過。

最大長度 = 4
```

## 關鍵點 / 易錯陷阱

- 必須先用 `set` 去重 + 加速查詢。
- **「只從起點展開」**才是 O(n) 的關鍵；少了這個檢查就退化成 O(n²)。
- 邊界：空陣列回傳 0。
- 重複元素不增長度（去重後處理）。
