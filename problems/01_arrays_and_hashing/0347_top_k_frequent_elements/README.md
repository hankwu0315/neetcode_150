# 347. Top K Frequent Elements

**Difficulty**: Medium  **URL**: https://leetcode.com/problems/top-k-frequent-elements/

## 題目描述（中文）

給定一個整數陣列 `nums` 和整數 `k`，回傳出現頻率**前 k 高**的元素（順序不限）。  
答案保證唯一。

**進階**：演算法時間複雜度需優於 O(n log n)（n 為陣列長度）。

## 範例 / 限制

**範例 1**
```
Input:  nums = [1,1,1,2,2,3], k = 2
Output: [1,2]
```

**範例 2**
```
Input:  nums = [1], k = 1
Output: [1]
```

**限制**
- `1 <= nums.length <= 10⁵`
- `-10⁴ <= nums[i] <= 10⁴`
- `1 <= k <= 不同元素個數`

## 解法

### 解法一：完整排序
先計頻、再依頻率排序，取前 k。
- **時間複雜度**：O(n log n)
- **空間複雜度**：O(n)

### 解法二：最小堆（Min-Heap）
維護大小為 k 的最小堆，存 `(freq, num)`；新元素若 `freq` 大於堆頂則替換。
- **時間複雜度**：O(n log k)
- **空間複雜度**：O(n + k)

### 最佳解：桶排序（Bucket Sort）

**核心思路**：
1. 計頻得到 `count: dict[num] = freq`。
2. 由於頻率最高就是 n（陣列長度），建立大小為 `n + 1` 的 buckets，`buckets[f]` 是「頻率 = f 的所有元素」。
3. 從 `f = n` 倒序往下掃，把元素放入結果，蒐集到 `k` 個即停止。

頻率範圍有界，桶排序避免比較排序，整體線性。

- **時間複雜度**：O(n)
- **空間複雜度**：O(n)

## 範例追蹤（最佳解）

```
nums = [1,1,1,2,2,3], k = 2

count = {1:3, 2:2, 3:1}
n = 6, buckets = [[], [3], [2], [1], [], [], []]   (索引 = 頻率)

從高頻往低掃：
f=6,5,4 → 空
f=3 → 找到 1，加入結果 [1]
f=2 → 找到 2，加入結果 [1,2] → 已收滿 k=2，結束

結果：[1, 2] ✓
```

## 關鍵點 / 易錯陷阱

- 桶的大小要 `n + 1`（索引 0 到 n 都需存在）。
- 同一桶可能有多個元素（相同頻率）。
- 若 k 很小，最小堆 O(n log k) 也是不錯選擇，且程式碼較短。
- Python 可用 `collections.Counter().most_common(k)`，但這違背「不直接排序」的進階要求。
