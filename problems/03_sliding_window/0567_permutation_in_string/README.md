# 567. Permutation in String

**Difficulty**: Medium  **URL**: https://leetcode.com/problems/permutation-in-string/

## 題目描述（中文）

給定兩個字串 `s1` 與 `s2`，判斷 `s2` 是否包含 `s1` 的某個**排列**作為子字串。  
換言之：是否存在 `s2` 的子字串，其字元計數與 `s1` 相同。

## 範例 / 限制

**範例 1**
```
Input:  s1 = "ab", s2 = "eidbaooo"
Output: true   ("ba" 為 s1 的排列)
```

**範例 2**
```
Input:  s1 = "ab", s2 = "eidboaoo"
Output: false
```

**限制**
- `1 <= s1.length, s2.length <= 10⁴`
- 全為英文小寫字母

## 解法

### 解法一：對每個窗口排序比對
枚舉 `s2` 中長度 `len(s1)` 的子字串，排序後比較。
- **時間複雜度**：O(n · k log k)
- **空間複雜度**：O(k)

### 解法二：每窗口建立計數比較
枚舉子字串，用計數陣列比 26 元素。
- **時間複雜度**：O(n · 26)
- **空間複雜度**：O(1)

### 最佳解：固定窗口 + 增量維護計數 + matches 變數

**核心思路**：

設 `s1` 長度為 `k`。維護兩個 26 元計數 `c1`（不變）與 `c2`（窗口內），以及變數 `matches = c1[i] == c2[i] 的字母數`：

1. 初始化 `c1, c2`（先放前 `k` 個字元到 `c2`），計算 `matches`。
2. 滑動窗口：
   - 加入 `s2[r]`：更新 `c2`，依變動修正 `matches`（變化前後若與 `c1[idx]` 相等狀態翻轉則 ±1）。
   - 移出 `s2[l]`：同理。
3. 若 `matches == 26` 表示計數完全相同 → `true`。

每步只 O(1)。

- **時間複雜度**：O(n)
- **空間複雜度**：O(1)

## 範例追蹤（最佳解）

```
s1="ab" → c1=[a:1,b:1,其他:0]
s2="eidbaooo", k=2

r=0..1 初始化窗口 "ei": c2=[e:1,i:1] 
  matches = (a:0==1?否) + (b:0==1?否) + (e:1==0?否) + (i:1==0?否) + 22 個 0==0 → 22

r=2 'd': 加入 → c2[d]=1. 變化前 c2[d]=0==c1[d]=0 (相等)，變化後 1!=0 (不等) → matches-=1 → 21
        移出 s2[0]='e': 變化前 c2[e]=1!=0，變化後 0==0 → matches+=1 → 22
r=3 'b': 加入 → c2[b]=1. 變化前 0==1?否, 變化後 1==1?是 → matches+=1 → 23
        移出 s2[1]='i': 變化前 1==0?否, 變化後 0==0?是 → matches+=1 → 24
r=4 'a': 加入 → c2[a]=1. 變化前 0==1?否, 變化後 1==1?是 → matches+=1 → 25
        移出 s2[2]='d': 變化前 1==0?否, 變化後 0==0?是 → matches+=1 → 26
matches==26 → true ✓
```

## 關鍵點 / 易錯陷阱

- 只用一個 `matches` 變數可避免每步 O(26) 比較。
- 變化判定要小心：「變動前是否相等」與「變動後是否相等」對 `matches` 的貢獻互相抵消或相加。
- `len(s1) > len(s2)` 直接回傳 `false`。
- 替代寫法：簡單 O(26n) 版本（每次比 `c1 == c2`），易理解但稍慢。
