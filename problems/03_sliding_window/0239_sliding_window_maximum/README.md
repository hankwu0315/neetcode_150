# 239. Sliding Window Maximum

**Difficulty**: Hard  **URL**: https://leetcode.com/problems/sliding-window-maximum/

## 題目描述（中文）

給定整數陣列 `nums` 與整數 `k`。長度為 `k` 的窗口由左到右滑過陣列，回傳每個窗口的**最大值**陣列。

## 範例 / 限制

**範例 1**
```
Input:  nums = [1,3,-1,-3,5,3,6,7], k = 3
Output: [3,3,5,5,6,7]
```

**範例 2**
```
Input:  nums = [1], k = 1
Output: [1]
```

**限制**
- `1 <= nums.length <= 10⁵`
- `-10⁴ <= nums[i] <= 10⁴`
- `1 <= k <= nums.length`

## 解法

### 解法一：每窗口取最大
直接掃描每個窗口取 max。
- **時間複雜度**：O(n · k)
- **空間複雜度**：O(1)

### 解法二：最大堆（max-heap）
推入 `(-value, index)`；取頂時若超出窗口則丟棄。
- **時間複雜度**：O(n log n)
- **空間複雜度**：O(n)

### 最佳解：單調遞減 Deque

**核心思路**：

維護一個**雙端佇列 `dq`** 儲存索引，使對應 `nums` 值遞減：

1. 對每個 `r`：
   - 移除 `dq` 中**不在窗口** 的隊首：`dq[0] <= r - k` → `popleft`。
   - 從 `dq` 尾部移除小於等於 `nums[r]` 的值（它們不可能再是最大）。
   - 加入 `r`。
2. 當 `r >= k - 1`：`dq[0]` 就是當前窗口最大值的索引。

每個元素至多進出 deque 一次。

- **時間複雜度**：O(n)
- **空間複雜度**：O(k)

## 範例追蹤（最佳解）

```
nums = [1,3,-1,-3,5,3,6,7], k=3

r=0 (1): dq=[0]               (窗口未滿)
r=1 (3): 1<=3 → 彈 0; dq=[1]   (窗口未滿)
r=2 (-1): -1<3 → 不彈; dq=[1,2], 答案 nums[1]=3
r=3 (-3): -3<-1 → 不彈; dq=[1,2,3], 1 == r-k(0) 不需彈; 答案 nums[1]=3
r=4 (5): -3,-1,3 都 ≤5 → 全彈; dq=[4]; 答案 5
r=5 (3): 3≤?5 不彈 (但等於要彈嗎? "≤" 才彈所以 3<5 不彈); dq=[4,5]; 答案 5
r=6 (6): 5≤6 彈; 3≤6 彈; dq=[6]; 答案 6
r=7 (7): 6≤7 彈; dq=[7]; 答案 7

[3, 3, 5, 5, 6, 7] ✓
```

## 關鍵點 / 易錯陷阱

- 用**索引**而非值：方便判斷是否還在窗口內。
- 從尾部彈出時用 `<=`（嚴格較小一定要彈，等於亦可彈出，不影響答案）。
- 隊首索引若 `<= r - k` 表示已離開窗口。
- 答案開始記錄的時機：`r >= k - 1`。
