# 0025. Reverse Nodes in k-Group

- 難度：Hard
- LeetCode：https://leetcode.com/problems/reverse-nodes-in-k-group/

## 題目描述（中文）

每 `k` 個一組，將串列就地反轉每一組；若最後不足 k 個則保持原序。
要求 O(1) 額外空間（不可改變節點值）。

## 範例 / 限制

- `[1,2,3,4,5], k=2` → `[2,1,4,3,5]`
- `[1,2,3,4,5], k=3` → `[3,2,1,4,5]`
- `[1,2,3,4,5,6], k=3` → `[3,2,1,6,5,4]`

## 解法

### 解法（最佳解）：分組反轉，pre-group tail 串接

1. 用 dummy。`group_prev` 為上一組的尾。
2. 重複：找第 k 個節點 `kth`（不足則停止）。
3. 反轉 [group_prev.next ... kth]，並將兩端正確接回。
4. 移動 `group_prev` 到剛反轉組的新尾。

關鍵：每次反轉只負責本組區段；反轉前先記下「組末的下一個」`group_next = kth.next`，作為下一輪的反轉終止。

- 時間：O(n)
- 空間：O(1)

## 範例追蹤

`[1,2,3,4,5], k=2`：
- 第 1 組：1,2 → 反轉成 2,1。dummy→2→1→3→4→5。
- 第 2 組：3,4 → 反轉成 4,3。dummy→2→1→4→3→5。
- 第 3 組：剩 1 個（5）< k → 結束。最終 `[2,1,4,3,5]` ✓。

## 關鍵點 / 易錯陷阱

- 找第 k 個節點時若不足 k 個 → **不反轉**直接返回。
- 反轉的「終止條件」是當下一個節點 == `group_next`（下一組起點），不要寫成 `cur != None`。
- 別忘了把上一組尾的 `next` 接到新頭，並把舊頭的 `next` 接到 `group_next`。
