# 0287. Find the Duplicate Number

- 難度：Medium
- LeetCode：https://leetcode.com/problems/find-the-duplicate-number/

## 題目描述（中文）

陣列 `nums` 長度 `n + 1`，元素值範圍 `[1, n]`，**僅有一個值重複**（可能出現多次）。
找出該重複值。要求：不可修改陣列、O(1) 空間、優於 O(n²)。

## 範例 / 限制

- `[1,3,4,2,2]` → `2`
- `[3,1,3,4,2]` → `3`
- `[1,1]` → `1`

## 解法

### 解法 1：HashSet（O(n) 空間，違反要求）

掃描存集合，第一個重複即答案。

### 解法 2：原地標記（修改陣列，違反要求）

把 `nums[abs(nums[i]) - 1]` 取負；若該位已負則找到重複。

### 解法 3：Floyd 找環起點（最佳解）

把 `i -> nums[i]` 視為函式形成的圖。值範圍 `[1, n]`，索引從 0；自 `i = 0` 出發必走入環，環的入口即為**重複值**（鴿籠原理：n+1 個值映到 n 個格子）。

兩階段：
1. 快慢指標相遇於環內。
2. `slow = 0`，兩者一步一步同走，再次相遇即環入口 → 答案。

- 時間：O(n)
- 空間：O(1)
- 不修改陣列。

## 範例追蹤

`[1,3,4,2,2]`，序列：`0→1→3→2→4→2→4→…`
- 階段 1：相遇於某點。
- 階段 2：自 0 與相遇點各走 1 步，會在 `2` 相遇。答案 `2` ✓。

## 關鍵點 / 易錯陷阱

- `n + 1` 個元素映到 `[1, n]` 必有環（與索引 0 相連），環入口即重複值。
- 二分版本（按值二分，計算 `<= mid` 個數）也是 O(n log n) 候選方案。
