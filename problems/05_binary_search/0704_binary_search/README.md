# 0704. Binary Search

- 難度：Easy
- LeetCode：https://leetcode.com/problems/binary-search/

## 題目描述（中文）

給定升序排序的整數陣列 `nums` 與目標值 `target`，回傳 `target` 的索引；若不存在回傳 `-1`。必須使用 **O(log n)** 演算法。

## 範例 / 限制

- `nums = [-1,0,3,5,9,12], target = 9` → `4`
- `nums = [-1,0,3,5,9,12], target = 2` → `-1`
- 所有元素互異；1 ≤ n ≤ 10^4。

## 解法

### 解法 1：迭代二分搜尋（最佳解）

`l = 0, r = n - 1`，每輪取 `mid = (l + r) // 2`：
- `nums[mid] == target` → 回傳 `mid`。
- `nums[mid] < target` → 答案在右半 → `l = mid + 1`。
- `nums[mid] > target` → 答案在左半 → `r = mid - 1`。
- `l > r` 時結束 → `-1`。

防止溢位寫法：`mid = l + (r - l) // 2`。

- 時間：O(log n)
- 空間：O(1)

### 解法 2：遞迴二分搜尋

語意相同，遞迴傳遞 `(l, r)`。空間 O(log n)（呼叫堆疊）。

## 範例追蹤（解法 1）

`nums = [-1,0,3,5,9,12], target = 9`

| l | r | mid | nums[mid] | 動作 |
|---|---|---|---|---|
| 0 | 5 | 2 | 3 | `<9` → l=3 |
| 3 | 5 | 4 | 9 | 命中 → 回傳 4 |

## 關鍵點 / 易錯陷阱

- 區間定義要一致：上面用 **閉區間 `[l, r]`** + `r = mid - 1`，迴圈條件 `l <= r`。若改用半開 `[l, r)` 則 `r = mid`、條件 `l < r`。混用會出 bug。
- `mid = (l + r) // 2` 在多數語言對大數會溢位，建議寫 `l + (r - l) // 2`。
