# 0153. Find Minimum in Rotated Sorted Array

- 難度：Medium
- LeetCode：https://leetcode.com/problems/find-minimum-in-rotated-sorted-array/

## 題目描述（中文）

升序、互異的陣列被旋轉了未知次數（例：`[0,1,2,4,5,6,7]` → `[4,5,6,7,0,1,2]`）。
回傳其最小值。要求 O(log n)。

## 範例 / 限制

- `[3,4,5,1,2]` → `1`
- `[4,5,6,7,0,1,2]` → `0`
- `[11,13,15,17]`（未旋轉）→ `11`
- 元素互異；1 ≤ n ≤ 5000。

## 解法

### 解法 1：二分（最佳解）

維持 `l, r`，比較 `nums[mid]` 與 `nums[r]`：
- `nums[mid] > nums[r]`：最小值在右半（旋轉點還沒到）→ `l = mid + 1`。
- `nums[mid] < nums[r]`：最小值在左半（含 mid）→ `r = mid`。

迴圈直到 `l == r`，回傳 `nums[l]`。

> 為何比 `nums[r]` 而不是 `nums[l]`？因為與 r 比可以正確處理「未旋轉」的情況（mid 一定 ≤ r，會走 `r = mid`，最終收斂到 0）。

- 時間：O(log n)
- 空間：O(1)

### 解法 2：暴力 O(n)

直接 `min(nums)`，作為對照。

## 範例追蹤（解法 1）

`nums = [4,5,6,7,0,1,2]`，l=0, r=6
| l | r | mid | nums[mid] | nums[r] | 動作 |
|---|---|---|---|---|---|
| 0 | 6 | 3 | 7 | 2 | 7>2 → l=4 |
| 4 | 6 | 5 | 1 | 2 | 1<2 → r=5 |
| 4 | 5 | 4 | 0 | 1 | 0<1 → r=4 |
| 4 | 4 |   |   |   | 結束 → nums[4]=0 ✓ |

## 關鍵點 / 易錯陷阱

- 比較對象選 **`nums[r]`**；用 `nums[l]` 在未旋轉時會出錯。
- `r = mid`（不是 `mid - 1`），因為 mid 本身可能就是最小值。
- 元素互異是條件；若有重複需改題（154）。
