# 0150. Evaluate Reverse Polish Notation

- 難度：Medium
- LeetCode：https://leetcode.com/problems/evaluate-reverse-polish-notation/

## 題目描述（中文）

給定後綴運算式（Reverse Polish Notation）`tokens`（字串陣列），求其結果。
合法運算子為 `+`、`-`、`*`、`/`，運算元為整數。
除法 **向 0 截斷**（不是向下取整）。

## 範例 / 限制

- `tokens = ["2","1","+","3","*"]` → `9`（`(2+1)*3`）
- `tokens = ["4","13","5","/","+"]` → `6`（`4 + 13/5`）
- `tokens = ["10","6","9","3","+","-11","*","/","*","17","+","5","+"]` → `22`
- 1 ≤ tokens.length ≤ 10^4

## 解法

### 解法 1：堆疊 — 標準解

掃描每個 token：
- 若是數字 → 推入堆疊。
- 若是運算子 → 彈出右運算元 `b`、左運算元 `a`，計算 `a op b` 後推回堆疊。

最終堆疊唯一元素即為結果。

**除法向 0 截斷**：Python 的 `//` 是向下取整，對負數會錯（例如 `-1 // 2 == -1`，但題目要 `0`）。需用 `int(a / b)` 或 `int(operator.truediv(a, b))`。

- 時間：O(n)
- 空間：O(n)

## 範例追蹤（解法 1）

`["2","1","+","3","*"]`：
| token | 動作 | 堆疊 |
|----|----|----|
| `2` | push | `[2]` |
| `1` | push | `[2, 1]` |
| `+` | pop b=1, a=2, push 3 | `[3]` |
| `3` | push | `[3, 3]` |
| `*` | pop b=3, a=3, push 9 | `[9]` |

結果 `9` ✓

## 關鍵點 / 易錯陷阱

- **除法向 0 截斷**：用 `int(a / b)` 而不是 `a // b`。例：`6 // -132 == -1`，但 `int(6 / -132) == 0`（正確）。
- 運算元順序：先 pop 的是右運算元 `b`，後 pop 的是左運算元 `a`，計算 `a op b`。
- 區分 token 是運算子還是數字：直接 `if t in {"+", "-", "*", "/"}`。
