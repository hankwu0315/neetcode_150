# 0853. Car Fleet

- 難度：Medium
- LeetCode：https://leetcode.com/problems/car-fleet/

## 題目描述（中文）

`target` 公里外有一個目的地。`n` 輛車在 0 公里處朝目的地行駛，每輛車的位置為 `position[i]`、速度為 `speed[i]`（單位 km/h）。

當較快的車追上較慢的車後，會以較慢車的速度同行（合併為「車隊」）。當車隊的所有車同時到達目的地，視為一個車隊。

求最終會有幾個車隊抵達 `target`。

## 範例 / 限制

- `target = 12, position = [10,8,0,5,3], speed = [2,4,1,1,3]` → `3`
- `target = 10, position = [3], speed = [3]` → `1`
- `target = 100, position = [0,2,4], speed = [4,2,1]` → `1`
- 1 ≤ n ≤ 10^5；0 ≤ position[i] < target

## 解法

### 解法 1：依位置排序 + 堆疊（最佳解）

關鍵觀察：
- 將車按 `position` **由大到小**（離終點近的在前）排序。
- 計算每輛車到達終點所需時間 `t = (target - position) / speed`。
- 從最前面的車開始掃描，維持「目前最大時間 max_time」：
  - 如果當前車的 `t > max_time`：它無法追上前面，自成一個車隊；更新 `max_time = t`。
  - 否則：它會追上前面（因為它在後但更快），合併入既有車隊，**不增加數量**。

最終車隊數即答案。

- 時間：O(n log n)（排序）
- 空間：O(n)（暫存 (pos, t) 配對）

### 解法 2：明確堆疊（教學用）

排序後逐一將時間 push 到堆疊；新時間若 ≤ 堆疊頂端，視為合併（不 push）。最終 `len(stack)` 為答案。本質與解法 1 相同。

## 範例追蹤（解法 1）

`target = 12, position = [10,8,0,5,3], speed = [2,4,1,1,3]`

依 position 由大到小排序：
| pos | speed | 到達時間 (12-pos)/speed |
|---|---|---|
| 10 | 2 | 1.0 |
| 8 | 4 | 1.0 |
| 5 | 1 | 7.0 |
| 3 | 3 | 3.0 |
| 0 | 1 | 12.0 |

掃描（max_time 初始 0）：
- 10 → t=1.0 > 0 → fleet=1, max=1.0
- 8 → t=1.0 ≤ 1.0 → 合併（同時到）
- 5 → t=7.0 > 1.0 → fleet=2, max=7.0
- 3 → t=3.0 ≤ 7.0 → 合併（追上）
- 0 → t=12.0 > 7.0 → fleet=3, max=12.0

答案 = `3` ✓

## 關鍵點 / 易錯陷阱

- 必須**從前往後**（最靠近終點的車優先處理），這樣後車「能否追上」就看時間是否 ≤ 前車。
- 比較條件：當前 `t ≤ max_time` 即合併（同時到也算一隊）。
- 浮點比較不會出問題，因為差距夠大；若擔心可改為比較分數 `(target-pos)*s_other` 與 `(target-pos_other)*s`。
