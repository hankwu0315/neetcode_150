# 0084. Largest Rectangle in Histogram

- 難度：Hard
- LeetCode：https://leetcode.com/problems/largest-rectangle-in-histogram/

## 題目描述（中文）

給定 `n` 個非負整數 `heights`，每個代表寬度為 1 的長條圖直條高度。求其中可圍成最大矩形的面積。

## 範例 / 限制

- `[2,1,5,6,2,3]` → `10`（高 5、寬 2 的矩形：indices 2,3）
- `[2,4]` → `4`
- `[1]` → `1`
- `[2,1,2]` → `3`（高 1、寬 3）
- 1 ≤ n ≤ 10^5；0 ≤ heights[i] ≤ 10^4

## 解法

### 解法 1：暴力 O(n^2)

對每個 `i` 向左、向右找出第一個比它矮的位置，計算寬度乘以 `heights[i]` 取最大。

- 時間：O(n^2)
- 空間：O(1)
- n=10^5 會 TLE。

### 解法 2：單調遞增堆疊（最佳解）

堆疊存「索引」，並維持其對應高度為**單調遞增**。

對 `i = 0..n`：
- 將 `cur = heights[i]`（`i = n` 時視為 0，作為 sentinel 強制清空堆疊）。
- 當堆疊非空且 `heights[stack[-1]] > cur`：彈出 `top`，
  此時 `heights[top]` 為矩形高，**寬度** 為：
  - 若堆疊空：`i`（從 0 開始一路延伸到 i-1）
  - 否則：`i - stack[-1] - 1`（左界為新 top 的右一格，右界為 i-1）
  
  以此計算面積取 max。
- 將 `i` 推入堆疊。

每個索引最多進出堆疊各一次。

- 時間：O(n)
- 空間：O(n)

> **NeetCode 風格替代寫法**：堆疊存 `(start_index, height)`，遇到較小高度時彈出並用「彈出時得到的最左 start」當寬度起點。等價且常見於課程影片。下方 Python 提供此版作為對照（`brute` 即 O(n^2)）。

## 範例追蹤（解法 2）

`heights = [2,1,5,6,2,3]`，加 sentinel 後等同遍歷 i = 0..6（h[6]=0）。

| i | h | 堆疊操作 | stack（索引） | 計算 | 最大面積 |
|---|---|----|---|---|---|
| 0 | 2 | push | [0] |  | 0 |
| 1 | 1 | h[0]=2>1 → pop 0；空 → 寬=1，面積=2*1=2；push 1 | [1] | 2 | 2 |
| 2 | 5 | push | [1,2] |  | 2 |
| 3 | 6 | push | [1,2,3] |  | 2 |
| 4 | 2 | h[3]=6>2 → pop 3；寬=4-2-1=1，6*1=6；h[2]=5>2 → pop 2；寬=4-1-1=2，5*2=**10**；push 4 | [1,4] | 10 | 10 |
| 5 | 3 | push | [1,4,5] |  | 10 |
| 6 | 0 | h[5]=3>0 → pop 5；寬=6-4-1=1，3*1=3；h[4]=2>0 → pop 4；寬=6-1-1=4，2*4=8；h[1]=1>0 → pop 1；空 → 寬=6，1*6=6 | [] |  | **10** |

答案 `10` ✓

## 關鍵點 / 易錯陷阱

- **sentinel**：在末尾追加高度 0（或迴圈到 i=n 時視 cur=0），可省去結束後再清空堆疊的特例。
- **寬度公式**：彈出 `top` 後，左界為「新堆疊頂端」+1（若空則為 -1），右界為 `i-1`，寬 = `i - stack[-1] - 1` 或 `i`（堆疊空時）。
- 堆疊維持嚴格遞增；相等高度不彈出（也可彈出，因為相同寬度面積會在後一輪算到）。但必須 `>` 不能 `>=` 才能避免重複。實作上 `>` 與 `>=` 都會得到正確最大值（差只在計算次數），但 `>` 更精簡。
